# Microsoft Planner / Project Ecosystem

## 2025–2026 structural change (SECONDARY, strong consensus)
- **Project for the web** consolidated into **Microsoft Planner**  
- Planner spans lightweight tasks → structured project (plans with Gantt/resources on higher SKUs)  
- **Project Online** retirement path called out into 2026 in partner blogs — verify on MS roadmap before citing dates in sales decks  
- Desktop **Project** still exists for heavy schedulers on higher plans  

## SKUs (as commonly described)
| Plan | Includes |
|------|----------|
| Planner Plan 1 | Basic tasks/scheduling |
| Planner & Project Plan 3 | Gantt, resources, desktop client access |
| Planner & Project Plan 5 | Portfolio / enterprise resource features |

## AI / Copilot
- Copilot in M365: plans, risk brainstorm, status decks, Excel variance  
- **Project Manager agent** (preview/rollout narratives): build plans, assign tasks, appear as team member, workback schedules, pull decisions from Teams transcripts  
- Requires Copilot licensing — not free with base Planner  
- Classic desktop Project Copilot timeline was still murky in early 2026 Q&A (UNVERIFIED future)

## GUI / UX
- Teams-native tabs; People view; Board/Grid/Schedule/Charts  
- Familiar Office DNA — lowest training for Microsoft shops  
- Not construction-native (no RFI/submittal/BOQ)

## Backend
- Microsoft 365 graph data; Dataverse heritage for Project for the web era  
- Deep Teams/Outlook/Loop coupling  

## Takeaway for Tornix-like
Competes for **generic task layer** inside enterprises already paying M365. Differentiator must be construction controls + Arabic PMO + agents on **project controls data**, not another Planner.
