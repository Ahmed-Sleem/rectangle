## Positioning

Single source of truth from **design → done**. Wins when teams live in Revit/AutoCAD/BIM. Less “pure GC financial OS” than Procore for some buyers; stronger model coordination.

## Product / workflow organization

Official workflow pillars (homepage):
1. **Artificial Intelligence** — automate workflows, surface insights  
2. **Document Management** — centralize design→ops docs  
3. **Bid Management** — network + bid packaging  
4. **Model Coordination** — clash detection, 3D issues  
5. **Project Management** — RFIs, submittals, daily reports, configurable  

### Historical product names (still appear in market)
- Autodesk Build (field + PM)  
- Autodesk Docs (document management)  
- BIM Collaborate / Coordinate  
- Takeoff  
- PlanGrid (acquired; drawing/field DNA absorbed)  
- Construction IQ (risk ML)  

## GUI / UX patterns

- **Product family** feel — users may bounce Docs vs Build vs Model tools (SECONDARY: “split across apps”)  
- **Model-centric issue tracking**: pin issues to BIM coordinates (differentiator vs 2D-only tools)  
- **Sheet/viewing** heritage from PlanGrid: offline drawings, markups  
- **Assistant** natural language over documents  
- Dense AEC professional UI; steeper for pure trade crews  
- Mobile solid but large models lag (SECONDARY)

### UX lessons
1. Deep BIM is a moat — expensive to clone; integrate viewers first.  
2. Avoid multi-app fragmentation in IA.  
3. Offline drawing access remains table stakes.

## AI capabilities
- Construction IQ: ML risk on projects (historical Autodesk strength)  
- Autodesk Assistant: NL queries (stronger on documents than live financials per some 2026 comparisons)  
- Spec/submittal automation themes  
- Ongoing Autodesk AI branding across Forma

## Strengths
- Unmatched design-construction continuum for Autodesk shops  
- Document + model coordination  
- Issue traceability to model  
- Brand trust in AECO  

## Weaknesses
- Complexity / cost for simple builders  
- Financial depth vs Procore debated  
- Arabic/MENA localization not a flagship  
- Learning curve  

## vs Tornix
ACC is the **BIM gravity well**. A Tornix-like product should **integrate** (import sheets, link issues) rather than rebuild Navisworks. Tornix’s BOQ→schedule + PMO AI is a different wedge than model clash.
