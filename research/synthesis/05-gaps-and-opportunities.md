                 CONSTRUCTION DEPTH

**Sweet spot:** High AI polish × Medium-high construction/PMO depth × **Native Arabic RTL** × Mid-market price × Fast onboarding.

## 2. Gaps by persona

| Persona | Pain | Underserved by |
|---------|------|----------------|
| Saudi/Egyptian PM | AR UI + P6 + daily chaos | Procore (EN), P6 (UX), WhatsApp (no structure) |
| Owner’s engineer | Portfolio health + approvals | Fragmented Excel |
| Site engineer | Offline drawings + simple tasks | Heavy enterprise UIs |
| Planner | CPM truth + scenarios | Pretty SaaS without CPM |
| Exec | One brief, not 17 logins | Tool sprawl |
| SMB contractor | Price + simplicity | Procore ACV |

## 3. Tornix claims vs market reality (opportunity validation)

| Claim | Market check |
|-------|----------------|
| AI foundation not feature | Valid direction — Helix/Plane prove it |
| BOQ→schedule | Real category (ALICE/Togal) — hard scientifically |
| Zero Arabic competitors | **False**; regional tools exist — win on *quality* of RTL+AI |
| <7 day launch | Credible for SMB config; not for enterprise processes |
| Beats P6 on CPM | Unlikely early — **interop** smarter than replace |
| Full +17 modules | Matches Procore breadth ambition — dangerous scope trap |

## 4. Strategic plays (pick one primary)

### Play A — “Arabic AI PMO for contractors” (recommended)
- Own Today brief, risks, EVM lite, Gantt, docs, approvals, chat  
- P6 import, WhatsApp, AR/EN  
- Integrate field depth over time  
- Price below Procore, above Zoho  

### Play B — “AI agent layer on existing stacks”
- Sit on Procore/P6/ACC via APIs  
- Faster to market; weaker system-of-record moat  

### Play C — “Owner capital portfolio + AI”
- Compete PMWeb/INGENIOUS  
- Longer sales cycles, higher ACV  

### Play D — “Fork/extend OSS”
- Plane + construction modules  
- GPL/open-core licensing constraints if OpenProject  

## 5. Moats worth building
1. **Bilingual construction ontology** (AR/EN terms, Saudi contract norms)  
2. **Agent audit + permission framework** trusted by enterprises  
3. **P6-quality import with opinionated execution layer**  
4. **WhatsApp bidirectional** structured messages  
5. **Templates library** for local project types (residential tower, hospital, road)  
6. **Data residency** KSA/UAE options  

## 6. Moats NOT worth early
- Full BIM clash engine  
- Hardhat CV platform  
- Global marketplace with 400 apps  
- Payroll/WPS (partner)  
- Desktop thick client  

## 7. Risk register for us as builders
| Risk | Mitigation |
|------|------------|
| Scope = second Procore | Strict package roadmap P0–P6 |
| AI vaporware trust | Citations + human approve |
| CPM correctness | Golden tests vs known schedules |
| GPL contamination | License review before copy |
| Mobile offline bugs | Sync protocol design early |
